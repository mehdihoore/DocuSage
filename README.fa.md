# داکیوسیج (DocuSage)

*یک ایجنت رایگان و بدون نیاز به سرور RAG که به شما امکان می‌دهد با کتابخانه و اسناد خود گفتگو کنید- از طریق رابط وب و تلگرام.*

[English 🇬🇧](./README.md) · [راهنمای نصب و استفاده](./GUIDE.md)

---

##  ایده اصلی توانایی گفتگو با اسنادیست که نمیتوان به سادگی آنها را خلاصه کرد. 
کار روی متونی نزدیکبه متون حقوقی که نیاز است بدون تفسیرهای خاص به هر بخشی از آن دسترسی یافت و متن اصلی را در کنار توضیح و توصیف و نه تفسیر گذاشت.

پس این ریپازیتوری سه بخش را به هم وصل می‌کند:

1. **یک پایگاه دانش** — اسناد شما، تکه‌تکه‌شده و تبدیل‌شده به بردار (embedding) و ذخیره‌شده در [AstraDB](https://www.datastax.com/products/datastax-astra) (یک پایگاه‌داده برداری سرورلس با پلن رایگان).
2. **یک چت‌بات مبتنی بر بازیابی (RAG)** — ساخته‌شده روی [Hono](https://hono.dev/) و اجراشده روی Cloudflare Workers، هم به‌صورت **وب‌اپ** و هم **ربات تلگرام** در دسترس است، و از Google Gemini استفاده می‌کند.
3. **یک پایپلاین استخراج داده** — بخشی که واقعاً فایل‌های PDF خام را به محتوای تمیز، ساختاریافته و قابل جست‌وجو تبدیل می‌کند.

کل این مجموعه طوری طراحی شده که فقط با پلن‌های رایگان کار کند. بدون سرور پولی، بدون دیتابیس پولی، بدون اشتراکی که یادتان برود لغوش کنید. اگر بتوانید یک اکانت Cloudflare، یک نمونه رایگان AstraDB و یک کلید API جِمینای بگیرید، می‌توانید این را اجرا کنید.

پشتیبانی از **فارسی** و متن راست‌به‌چپ — اما در سطح کد، هیچ‌چیز مخصوص فارسی نیست و برای هر زبانی قابل استفاده است.

## سه بخش اصلی

### ۱. چت‌بات (Cloudflare Worker + ربات تلگرام)

بخشی که کاربر واقعاً با کتابها حرف می‌زند.

- **رابط وب** — جست‌وجوی معنایی روی کتابخانه شما، به‌علاوه یک پنجره چت که نتایج جست‌وجوی برداری را با جست‌وجوی زنده وب ترکیب می‌کند تا پاسخ‌هایی مستندتر بدهد. از راست‌به‌چپ پشتیبانی می‌کند، مارک‌داون را درست رندر می‌کند و نشان می‌دهد پاسخ از کدام سند/بخش آمده است.
- **ربات تلگرام** — همان مغز، اما در جیب شما. دستور `/search` برای یک جست‌وجوی سریع وب و `/chat` برای یک گفت‌وگوی کامل RAG با پایگاه دانش شما.

هر دو از یک فایل ورکر مشترک و همان منطق بازیابی و تولید پاسخ استفاده می‌کنند.

### ۲. نوت‌بوک استخراج داده (`DocuSage_Data_Ingestion.ipynb`)

نوت‌بوک اصلی Colab برای وارد کردن اسناد به AstraDB: فایل‌های PDF/DOCX/TXT را بارگذاری می‌کند، به تکه‌های کوچک تقسیم می‌کند، با جِمینای امبد (embed) می‌کند، هر تکه را برچسب و خلاصه می‌کند و همه را به مجموعه برداری شما اضافه می‌کند. برای کتابخانه‌های سند عمومی مناسب است.

### ۳. 🆕 پایپلاین OCR و SQL (`docusage_ocr_sql_pipeline.py`)

بخش تازه اضافه شده که یک مشکل خاص‌تر و آزاردهنده‌تر را حل می‌کند: **تبدیل یک پوشه پر از کتاب‌های اسکن‌شده/PDF به یک کتابخانه تمیز، بدون تکرار و آماده برای پایگاه‌داده** — بدون این‌که مجبور باشید مدام بالای سرش بنشینید یا به‌خاطر یک تفاوت کوچک در نام فایل، یک کتاب را پنج بار پردازش کنید.

به زبان ساده، برای هر PDF پیدا می‌کند:

- **زبان کتاب را تشخیص می‌دهد** با OCR کردن فقط یک صفحه از وسط کتاب (ارزان، سریع، بدون هدر رفتن API روی کل کتاب).
- **OCR کامل را اجرا می‌کند** با موتور اسناد هوش‌مصنوعی Mistral، و فایل‌های PDF بزرگ را خودکار تکه‌تکه می‌کند تا روی یک اسکن هزارصفحه‌ای گیر نکند.
- **یک فایل مارک‌داون درست‌وحسابی می‌سازد** — تیترها لنگر (anchor) می‌گیرند، فهرست مطالب خودکار ساخته می‌شود و حروف فارسی/عربی در لینک‌های لنگر درست حفظ می‌شوند (نکته‌ای که اکثر ابزارهای فهرست‌سازی خودکار آن را خراب می‌کنند).
- **یک فایل `.docx` معادل هم می‌سازد**، تا یک نسخه انسان‌پسند هم داشته باشید، نه فقط مارک‌داون.
- **متادیتا را با یک مدل زبانی استخراج می‌کند** — عنوان (لاتین و فارسی)، نویسنده، مترجم، ناشر، تاریخ، یک توضیح کوتاه، و حتی برچسب‌ها/دسته‌بندی به سبک سئو — همه به‌صورت JSON ساختاریافته. اگر یک اثر فلسفی ترجمه‌شده به‌صراحت نام مترجم را نیاورده باشد، حتی تلاش می‌کند با دانش زمینه‌ای‌اش درباره مترجمان مشهور فارسی، حدس بزند مترجم چه کسی بوده است.
- **پیش از هر کار واقعی، پنج لایه بررسی تکراری بودن انجام می‌دهد**:
  ۱. نام فایل دقیقاً قبلاً در پایگاه‌داده وجود دارد
  ۲. هش فایل یکسان است (نسخه‌های تغییرنام‌یافته را می‌گیرد)
  ۳. اثر انگشت محتوا یکسان است (اسکن دوباره همان صفحات را می‌گیرد)
  ۴. یک بررسی معنایی با مدل زبانی، عنوان/نویسنده/مترجم را با همه کتاب‌های قبلاً پردازش‌شده مقایسه می‌کند (شبه‌تکرارهایی مثل «G.W.F. Hegel» در برابر «Georg Wilhelm Friedrich Hegel» را می‌گیرد)
  ۵. بررسی فایل‌های `.docx`/`.md` باقی‌مانده از اجرای قبلی روی دیسک
- **فایل SQL آماده برای ایمپورت می‌نویسد** — دستورات `INSERT` برای جدول `books`، جدول `book_contents` (یک ردیف برای هر بخش)، و جدول `wp_sync` اگر قصد انتشار در یک سایت وردپرس را دارید.
- **در برابر قطعی و محدودیت نرخ درخواست، مقاوم است** — بین چند کلید API جِمینای می‌چرخد و چند مدل مختلف را امتحان می‌کند و در نهایت اگر همه گزینه‌های جِمینای تمام شد، به Mistral برمی‌گردد. اگر وسط پردازش یک کتاب کرش کند، دفعه بعد آن را بی‌خودی دوباره پردازش نمی‌کند.

این اسکریپت قرار است در **Google Colab** اجرا شود، فایل‌های PDF را از یک پوشه گوگل‌درایو بخواند و خروجی مارک‌داون/DOCX/SQL را دوباره در درایو بنویسد. برای راهنمای کامل به [راهنما](./GUIDE.md) مراجعه کنید.

## فناوری‌های کلیدی

| بخش | کاربرد |
|---|---|
| **Hono** | فریم‌ورک سبک وب روی Cloudflare Workers |
| **Google Gemini** | امبدینگ، پاسخ‌های چت، استخراج متادیتا، ترجمه |
| **Mistral OCR** | تبدیل کتاب‌های اسکن‌شده/PDF به مارک‌داون ساختاریافته |
| **AstraDB** | پایگاه‌داده برداری سرورلس برای جست‌وجوی معنایی |
| **Cloudflare Workers AI** | جایگزین برای خلاصه‌سازی/استخراج کلیدواژه در صورت در دسترس نبودن جِمینای |
| **DuckDuckGo / Google Search** | جست‌وجوی عمومی وب داخل ربات تلگرام |
| **Tailwind CSS + markdown-it** | استایل رابط وب و رندر مارک‌داون |
| **PyMuPDF (`fitz`), python-docx** | تقسیم/بازرسی PDF و ساخت فایل `.docx` در پایپلاین |

## شروع سریع

نسخه کوتاه — برای نسخه کامل با توضیح همه متغیرهای محیطی، به [راهنما](./GUIDE.md) مراجعه کنید.

```bash
git clone https://github.com/mehdihoore/DocuSage
cd DocuSage
npm install
```

یک فایل `.dev.vars` با کلیدهای خودتان بسازید (جِمینای، AstraDB، تلگرام، جست‌وجوی گوگل) — **این فایل را هرگز کامیت نکنید**، باید از قبل در `.gitignore` باشد. سپس:

```bash
npx wrangler dev      # اجرای محلی برای تست
npx wrangler deploy   # استقرار روی Cloudflare
```

وب‌هوک ربات تلگرام‌تان را به آدرس دیپلوی‌شده وصل کنید، AstraDB را پر کنید (با نوت‌بوک، یا با پایپلاین OCR اگر با کتاب‌های اسکن‌شده کار می‌کنید) و کار تمام است.

## یک نکته درباره کلیدهای محرمانه

هر اطلاعات محرمانه‌ای که این پروژه با آن سروکار دارد — کلیدهای جِمینای، توکن‌های AstraDB، توکن ربات تلگرام، کلید Mistral — باید فقط در فایل محلی `.dev.vars` و در بخش متغیرهای محیطی داشبورد Cloudflare (برای نسخه نهایی) نگهداری شود. هرگز آن‌ها را در کامیت‌ها، ایشوها یا گفت‌وگوها کپی نکنید. اگر کلیدی جایی لو رفت، همان لحظه از پنل سرویس مربوطه باطلش کنید و یک کلید جدید بسازید.
